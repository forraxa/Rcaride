Esquema dataset mpg

  • manufacturer: Marca
  • model: Modelo
  • displ: Motor en litros
  • year: Año de fabricación
  • cyl: Número de cilindros
  • trans: Tipo de transmisión
  • drv: Tracción. f = frontal, r = trasera, 4 = 4 ruedas.
  • cty: Consumo en ciudad en millas/galón
  • hwy: Consumo en autopista en millas/galón
  • fl: Tipo de combustible/fuel
  • class: Tipo de coche

Esquema dataset diamonds

  • price: Precio en $
  • carat: Peso del diamante
  • cut: Calidad del corte. (Fair, Good, Very Good. Premium, Ideal)
  • color: Color del diamante
  • clarity: Claridad del diamante. (I1 (worst), SI2, SI1, VS2, VS1, VVS2, VVS1, IF (best))
  • x: Largo
  • y: Ancho
  • z: Profundo
  • depth: Porcentaje profundida total.
  • table: Ancho de la parte superior del diamante con relación al punto más ancho

ggplot + color + size + alpha

  • Color en función de class: Tipo de coche
  • Tamaño en función de cty: Consumo en ciudad
  • Transparencia en función de hwy: Consumo en autopista
ggplot(data = mpg) + 
  geom_point(mapping = aes(x = displ, y = hwy, color = class, size = cty, alpha = hwy))

ggplot + shape

  • Formas en función de class: Tipo de coche
    (En ggplot se pueden representar máximo 6 formas)
ggplot(data = mpg) + 
  geom_point(mapping = aes(x = displ, y = hwy, shape = class))

Las estéticas manuales

Las estéticas se pueden declarar manualmente declarandolas fuera de la estética.

  • color = nombre del color en formato string.
  • size = tamaño del punto en mm.
  • shape = formas del punto de 0 a 25.
  • fill = interior

ggplot(data = mpg) + 
  geom_point(mapping = aes(x = displ, y = hwy), color = "red", size = 3, shape = 24, fill = "yellow" )

FACETS

facet_wrap

Segmentar el gráfico en función de una variable.

  • Segmentar en función de class : Tipo de coche.
# facet_wrap(~<FORMULA_VARIABLE>) : la variable tiene que ser discreta  
ggplot(data = mpg) + 
  geom_point(mapping = aes(x = displ, y = hwy)) +
  facet_wrap(~class, nrow = 2)

facet_grid

Segmentar el gráfico en función de dos variables.

  • Segmentar en función de drv: Tipo de transmisión (eje y) y class : Tipo de coche (eje x).
# facet_grid(<FORMULA_VARIABLE>~<FORMULA_VARIABLE>) 
ggplot(data = mpg) + 
  geom_point(mapping = aes(x = displ, y = hwy)) +
  facet_grid(drv~class)

Diferentes de geometrías

  • linetype: Cada tipo de transmisión drv tiene distinta línea.
  • color: Cada tipo de transmisión drv tiene distinto color. (líneas y puntos)
ggplot(data = mpg) + 
  geom_point(mapping = aes(x = displ, y = hwy, color = drv)) +
  geom_smooth(mapping = aes(x = displ, y = hwy, linetype = drv, color = drv))

  • group: Realiza la estética agrupada por cada tipo de transmisión drv
ggplot(data = mpg) + 
  geom_smooth(mapping = aes(x = displ, y = hwy, group = drv, color = drv))

  • Las geometrías pueden tener declaradas las estéticas globales.
  • Se pueden sobreescribir las estéticas globales por unas locales.
  • filter: Filtrar los datos donde class: Tipo de coche sea “suv”.
  • se = F elimina el intervalo de confianza de smooth
ggplot(data = mpg, mapping = aes(x = displ, y = hwy)) + 
  geom_point(mapping = aes(color = class)) +
  geom_smooth(data = filter(mpg, class == "suv"))

Bars Charts

geom_col

  • El tamaño de las barras proviene de los valores. (usa stat_identity por defecto para dejar el dato como está)
ggplot(data = diamonds) + 
  geom_col(mapping = aes(x = price, y = depth))

geom_bars

Contar número de casos
  • El tamaño de las barras es prorcional al número de casos de cada grupo.
  • Por defecto tiene stat = “count”.
    (usa stat_count por defecto para contar el número de casos por cada posición)
ggplot(data = diamonds) + 
  geom_bar(mapping = aes(x = cut))

Representar x, y con identity

Uso de geom_bar con stat = “identity”.

Ejemplo con datos ya contabilizados.
* Se crea un dataset con tribble

demo_diamonds <- tribble(
  ~cut,       ~freqs,
  "Fair",       1610,
  "Good",       4906,
  "Very Good", 12082,
  "Premium",   13791,
  "Ideal",     21551
)

ggplot(data = demo_diamonds) + 
  geom_bar(mapping = aes(x = cut, y = freqs),
           stat = "identity")

Representar la proporción de casos
  • ..prop.. y group trabajan juntas.
  • group = 1 es necesario para contar cada registro individualmente.
ggplot(data = diamonds) + 
  geom_bar(mapping = aes(x = cut, y = ..prop.., group = 1))

Diagrama de barras apilado
  • Se representa el corte cut
  • Se colorea por la claridad, clarity, del diamante
ggplot(data = diamonds) + 
  geom_bar(mapping = aes(x = cut, fill = clarity))

Control de el apilado o stack
  • Control mediante position “fill”, “dodge”.
position = “fill”
  • Interesante para evaluar proporciones.
ggplot(data = diamonds, mapping = aes( x = cut, fill = clarity)) + 
  geom_bar( position = "fill")

position = “dodge”
  • Las barras una al lado de las otras.
ggplot(data = diamonds, mapping = aes( x = cut, fill = clarity)) + 
  geom_bar( position = "dodge")

Solución para el Overplotting

  • Uso de position = “jitter”
  • Añade ruido aleatorio para evitar la superposición.
ggplot(data = mpg, mapping = aes(x = displ, y = hwy)) + 
  geom_point(position = "jitter")

Sistema de coordenadas (rotar el gráfico)

  • coord_flip() intercambia x por y
ggplot(data = mpg, mapping = aes(x = class, y = hwy)) + 
  geom_boxplot()

ggplot(data = mpg, mapping = aes(x = class, y = hwy)) + 
  geom_boxplot() + 
  coord_flip()

Coordenadas Polares (tela araña)

  • coord_polar()
ggplot(data = diamonds) +
  geom_bar(mapping = aes(x= cut, fill = cut),
           show.legend = F,
           width = 1) +
  theme(aspect.ratio = 1) +
  labs(x = NULL, y = NULL) +
  coord_polar()

ggplot(data = mpg, mapping = aes(x = class, fill = trans)) + 
  geom_bar() +
